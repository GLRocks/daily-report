## 一、项目经理反思：为什么版本模板一直在变

### 1.1 根因分析

| 问题 | 根因 | 影响 |
|------|------|------|
| 模板格式漂移 | 缺乏**模板锁定机制**——模板文件虽名为`v12`且声明"冻结"，但每日生成时未强制执行diff校验 | CSS变量、板块顺序、类名被误改 |
| 内容时效性不足 | **搜索规则执行衰减**——`freshness=day`规则在cron payload中存在，但子代理执行时未强制检查 | GPT-4.1/GPT-5.5版本混淆、Kimi K2.5/K2.6混淆 |
| 数据准确性差 | **INTC股价事件**暴露系统性脆弱——API失败时子代理回退搜索引擎，返回旧价格 | $24.50 vs $120.60，差5倍 |
| 超链接失效 | `href="#"`在模板中作为占位符，生成时未替换为真实URL | 用户点击"播放原声"无反应 |
| 推送失败 | Cloudflare Tunnel随机URL vs GitHub Pages固定URL混淆 | 用户收到无效链接 |

### 1.2 核心问题：规则存在 ≠ 执行有效

你的系统有个致命矛盾：**规则文档极其完善，但执行层没有强制能力**。

- `source_map.md` — 写了13板块详细要求 ✓
- `content_freshness_protocol.md` — 写了6层强制机制 ✓  
- `pre_flight_check.py` — 写了自动化检查脚本 ✓
- `cron/jobs.json` — payload里塞了全部规则 ✓
- `accuracy_errors.md` — 记录了错误和修正 ✓

**但是**：
- 子代理生成内容时，不会逐条对照这些规则
- 没有预检脚本在**生成后**自动运行
- API失败时，没有"停止生成"的硬机制，只有"建议中止"的软提示
- 模板对比是人工的，不是自动diff

### 1.3 结论

模板一直在变，不是因为规则不够多，而是因为**没有执行层把规则变成强制约束**。

## 二、各板块资深专家评审

### S1 核心持仓实时行情 — 量化分析师评审

**严重问题**: 
- **INTC价格错误**: 网站显示$120.29（已修正），但此前推送过$24.50
- **日涨跌幅可疑**: AMD显示-1.74%，但ifind API的`pct_change_1m`约等于0%。真实日涨跌应搜索获取或标注"待确认"
- **YTD硬编码**: "YTD+240%"等数字无法验证是否为API获取或模板残留

**修复要求**:
1. 股票数据100%走ifind API → 写入CSV → 模板读取CSV
2. API任一失败 → 该股票标注"价格暂缺"，禁止搜索回填
3. 日涨跌幅：ifind不提供US stock日涨跌 → **删除此字段**，只保留收盘价+数据日期
4. YTD：如需显示，必须搜索获取+标注来源，否则删除

**信号**: 数据层可信度差，影响全部后续投资决策的基准。
