# 更新: 每次用户反馈错误时追加

## 格式
## YYYY-MM-DD
- **错误**: 具体描述
- **板块**: S{编号} {板块名}
- **根因**: 为什么发生
- **修正**: 已采取/将采取的修正措施
- **状态**: 已修正 / 待修正 / 观察中

## 2026-05-13
- **错误**: INTC股价显示$24.50，真实收盘价$120.60，差约5倍；全部20只股票的日涨跌幅均错误（真实±0.1%平盘，报告里±1-11%）
- **板块**: S1 核心持仓实时行情
- **根因**: 
  1. 价格：19/20来自ifind API（正确），但INTC的API调用失败/遗漏，子代理回退到搜索引擎，搜索返回2024年底旧价格（$24.50≈2024.12价格）
  2. 涨跌幅：ifind API的`pct_change_1m`是1分钟涨跌幅（≈0%），不是日涨跌幅。子代理使用搜索引擎获取日涨跌幅，但搜索结果不准确/可能是前一日数据/来源混杂
  3. 子代理未严格执行"API失败→中止/告警"规则，擅自用搜索回填
- **修正**:
  1. 股票数据获取流程重写：价格100%来自ifind API；API任一股票失败→中止报告生成，禁止搜索回填
  2. 日涨跌幅：ifind API不提供日涨跌幅（US stocks不支持close_summary）→方案A：去掉日涨跌幅，只保留价格和YTD；方案B：如需日涨跌，建立"搜索→至少2个来源交叉验证→不一致时标注'待确认'"机制
  3. 每日报告生成前运行stock validation脚本：API获取全部20只股票→写入CSV→报告模板从CSV读取
  4. cron payload追加："股票数据必须通过ifind API获取，禁止搜索引擎回填股价"
  5. 自检清单新增："[ ] 所有股票价格与ifind API实时数据一致？"
- **状态**: 待修正
