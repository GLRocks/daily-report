## Lesson Learned — Configuration Sync Protocol (CRITICAL)

When ANY format fix is applied, MUST update ALL 4 layers within the same commit:

1. **Output HTML** (daily_report_YYYY-MM-DD.html)
2. **Template HTML** (agentic_market_daily_template_v12.html) — the source of truth for next day
3. **Source Map** (source_map.md) — human-readable specification
4. **Pre-flight Check** (pre_flight_check.py) — machine-enforced validation

**If any layer is skipped, the fix will be lost on next generation.**

This is exactly what happened: I fixed the output HTML but didn't update the template or pre-flight checks, so the next day's cron would regenerate from the old template.

User explicitly called this out. Do not let this happen again.
